Cookie 简介 :

http 协议具有无状态的特性 , 而为了记录访问者的一些信息 ( 如登录状态等 ) , 通过 Cookie 和 Session 技术来实现记录访问者的一些基本信息

Cookie 是一种本地存储技术 , 是浏览器储存在用户的机器上的一个很小的文本文件。Cookie 是纯文本 , 没有可执行代码。储存一些服务器需要的信息 , 每次请求站点 , 会发送相应的cookie , 这些cookie可以用实现辨别用户身份信息等功能

Cookie 如何记录浏览器的登录状态 ( 会话跟踪 )

  • 浏览器向服务器发送登录请求 , 携带用户名和密码
  • 服务器进行登录信息的验证 , 如果成功 , 在给浏览器的响应中 , 设定了一个 Cookie ( 如 SID1234765432167 )
  • 页面跳转 , 发送的请求中携带了刚才服务器设定的 Cookie ( SID1234765432167 )
  • 服务器收到请求后 , 会从 Cookie 中取得 SID , 并验证是否为有效的 ID 值
  • 如果 ID 值有效

Cookie 的保存

document.cookie = "username=CuiJn"; //存Cookie
document.cookie = "password=123456"

document.cookie;    //取Cookie  格式为 "name1=value1; name2=value2; name3=value3"
//  这种方法会一次性去除全部 Cookie , 类型为字符串


// 取指定 Cookie ↓    需要将 Cookie 字符串进行两次切分实现
function getCookie(){
    var cookie = document.cookie;
    var arr = cookie.split("; ");
    var key_value;
    for(let i = 0; i < arr.length; i++){
        key_value = arr[i].split("=");
        if (key_value === "password") {
            return key_value;
        }
    }
    return null;
}

Cookie 选项

Cookie 路径

// path: 路径   当 Cookie 设定了路径 , 仅在这一文件夹下的 html 文件 , 可以访问此 Cookie  ( 包括其父级文件夹 ) 
// 在存 Cookie 时 , 如果没有设定 path  , 则默认设为当前 html 所在文件夹 

// 手动设定 path 方式 ↓

document.cookie = "name=value;path=url"

Cookie 有效期

// 如果系统时间超过了有效时间 , Cookie 会被浏览器删除
// 在没有设定有效时间的情况下 , 默认值为 session ( 浏览器关闭后结束 , 删除该 Cookie ) 
document.cookie = "name=value; expires=Date对象转成的字符串"

利用 Cookie 的有效期 , 可以实现 Cookie 的删除

let d = new Date();
d.setDate(d.getDate() - 1);
document.cookie = "neme=value; expires=" + d;

Cookie 的不可取代之处

Cookie 之所以不可取代 , 是因为它与服务器有联系

  • 浏览器在发送请求时 , 会自动将当前页面能够访问的 Cookie 发送给服务器
    通过请求头 cookie: aaa=bbb; ccc=ddd

  • 服务器同样有能力设定浏览器的 Cookie
    通过响应头 set-cookie: aaa=bbb;ccc=ddd
